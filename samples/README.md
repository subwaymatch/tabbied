# Made with Tabbied — 10 sample websites

Ten small, self-contained sample websites that use **Tabbied** generative
artworks as their primary design accent. Each site is themed end-to-end with a
single palette from the [Tabbied palette library](../lib/paletteLibrary.ts) and
built around one artwork from the [`packages/tabbied/artworks`](../packages/tabbied/artworks)
catalog — the same designs, palettes, and rendering pipeline that power
[tabbied.com](https://tabbied.com), baked into static HTML.

They exist to show what Tabbied looks like *in use*: as a hero backdrop, a
product-card texture, an editorial cover, a section divider, a full-bleed brand
motif.

## Open them

Start at [`index.html`](./index.html) for the gallery, or open any site's
`index.html` directly in a browser. Everything is static and self-contained —
`css-doodle` is vendored in [`assets/`](./assets), so the pages render offline
with no build step and no network calls (web fonts aside).

## How they're built

The HTML is generated by [`generate.mjs`](./generate.mjs). Regenerate with:

```bash
node samples/generate.mjs
```

The generator reads the **real** artwork JSON from
`packages/tabbied/artworks/` and applies the same option/color/size
substitution that the Tabbied engine does at runtime — see
[`lib/tabbied-embed.mjs`](./lib/tabbied-embed.mjs), a dependency-free port of
[`packages/tabbied/src/core/doodleSource.ts`](../packages/tabbied/src/core/doodleSource.ts).
Each design is emitted as the exact `<style>` + `<css-doodle use="var(--rule)">`
pair Tabbied mounts, with a fixed `seed` so patterns are reproducible.

Because the accents come straight from the artwork definitions and library
palettes, updating either upstream and re-running the generator reflows every
sample.
