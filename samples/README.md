# Made with Tabbied, 20 sample websites

Twenty small sample websites that use **Tabbied** generative artworks as their
primary design accent. Each site is a fictional brand themed end-to-end with a
single palette from the [Tabbied palette library](../lib/paletteLibrary.ts) and
built around one preset artwork from
[`packages/tabbied/artworks`](../packages/tabbied/artworks). Together they span
twenty palette/artwork pairings and show what Tabbied looks like *in use*, as a
hero backdrop, a product-card texture, an editorial cover, a full-bleed motif.

They come in two flavours that demonstrate the two ways to consume the library:

| Set | Count | How it renders | Lives in | Served at |
|-----|-------|----------------|----------|-----------|
| **Static HTML** | 10 | Embeds the `css-doodle` engine directly in plain HTML | `public/samples/` (built) | `/samples/…` |
| **React component** | 10 | Renders the `TabbiedArtwork` React component in Next.js routes | `app/showcase/<slug>/` | `/showcase/<slug>/` |

The whole set is browsable from the site header (**Showcase** →
[`/samples/`](/samples/)), a gallery that lists all twenty builds in two groups.
The gallery is itself a React route ([`app/samples/page.tsx`](../app/samples/page.tsx))
whose thumbnails are live `TabbiedArtwork` components, so nothing is stretched.

> Note on links: the site is a static export with `trailingSlash: true`, so a
> sample lives at `/samples/<dir>/`, the directory URL, **not**
> `/samples/<dir>/index.html` (that form 404s under trailing-slash hosting).

## Static HTML builds (`/samples/…`)

Like the React set, each static site is a full single-page layout (hero, about,
product grid, features, testimonials, closing band, newsletter, footer) and uses
**several** Tabbied artworks in one palette. Card imagery is left as an image
placeholder carrying a GPT Image 2 prompt (see below).

These are generated by [`generate.mjs`](./generate.mjs), a small data-driven
engine (four layouts, shared sections) that renders each artwork through
[`lib/tabbied-embed.mjs`](./lib/tabbied-embed.mjs), a dependency-free port of
[`packages/tabbied/src/core/doodleSource.ts`](../packages/tabbied/src/core/doodleSource.ts).
The decorative accents (hero, about panel, band) are sized and re-seeded by
[`assets/tabbied-runtime.js`](./assets/tabbied-runtime.js): it derives each
doodle's grid from its box so cells stay square (**never stretched**, a cover
fit) and shuffles the drawing on an interval. `css-doodle` and the runtime are
vendored into the output, so the pages render with no build step and no runtime
network calls (web fonts aside). The gallery is the React route above.
Regenerate with:

```bash
node samples/generate.mjs   # writes public/samples/<dir>/ + assets/
```

Output is committed under `public/samples/`, mirroring how
`scripts/optimize-images.mjs` writes into `public/images/`.

### Image placeholders

Product/card imagery is not a Tabbied artwork; it is a labelled placeholder that
shows a ready-to-use **GPT Image 2** prompt (also in a `data-image-prompt`
attribute). Each prompt has the site's color palette and background color
appended, so a generated image blends into the page. Generate an image and swap
the placeholder for an `<img>`. See
[gpt-image-2](https://developers.openai.com/api/docs/models/gpt-image-2).

## React component builds (`/showcase/<slug>/`)

Each of these is a full single-page layout that renders **several** Tabbied
artworks in one shared palette (one for the hero, another for the about panel,
another for the closing band), so a site shows off distinct patterns at once.
The decorative artworks re-seed on an interval (dynamic), and card imagery uses
the same GPT Image 2 placeholders described above.

These are real Next.js routes. Each `app/showcase/<slug>/page.tsx` imports its
handful of presets from `tabbied/artworks` (so the bundler ships only what it
uses) and passes them as a slug→definition map to
[`components/showcase/ShowcaseSite`](../components/showcase/ShowcaseSite.tsx), a
data-driven renderer that dresses one of four layouts (`split`, `spotlight`,
`editorial`, `boutique`) with the [`TabbiedArtwork`](../packages/tabbied/src/react/TabbiedArtwork.tsx)
component, cycling through the site's `artworks` list so each section shows a
different pattern. Content and theming live in
[`components/showcase/showcaseData.ts`](../components/showcase/showcaseData.ts);
each site's palette becomes CSS custom properties, so one stylesheet themes all
ten brands. They build with the rest of the site (`npm run build`).
